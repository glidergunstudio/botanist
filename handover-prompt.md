# Botanist（植物学者のガーデン）開発引き継ぎプロンプト

## プロジェクト概要

GPS連動のリアルワールド植物育成PWAゲーム。
プレイヤーは植物学者として街を歩いて種を拾い、プランターで育て、未知の植物を発見・命名する。

**リポジトリ**: botanist（GitHub）
**技術スタック**: HTML/JS PWA + Leaflet.js + Firebase（Auth/Firestore/Functions）+ IndexedDB

## ゲーム設計（確定済み）

### コアコンセプト
- 「何か分からない種を大切に育てて、何か分かったときの感動」
- 「植物を集めるコレクション性」
- 完全オリジナルの植物世界（現実の物理法則のみ共通）
- 開発者は植物を一切定義しない。全ての植物はプレイヤーが発見・命名する

### コアループ
```
GPS地図で種を発見 → 現地に行って拾う → スキルで種を鑑定
→ プランターに植える → 生育条件を管理 → 成熟・種を収穫
→ 再植え / 市場で売却 / 交配（将来フェーズ）→ 新種発見で命名
```

## 遺伝子システム（確定済み）

### 3層構造
```
層1: 遺伝子（loci）    → クライアントに公開。40座×2アレル（A/aのみ）
層2: タンパク質        → サーバー側のみ。wiring設計図から計算。15種
層3: 形質（phenotype）→ 結果だけクライアントに渡す
```

### 形質の2経路
```
経路1: 連続値 → タンパク質経由（草丈・花色・香り・味など）
経路2: 離散値 → アレル直接判定（草姿・根の形・花序・果実構造など）
```

### 40座の染色体配置
```
染色体1: 草姿（L00〜L07）
  L00・L01: 草丈（2座）
  L02: 茎の太さ
  L03: 頂端優勢
  L04: 節間の長さ
  L05: 重力屈性（直立←→匍匐）
  L06: 巻きひげの有無
  L07: 木化（草本←→木本）

染色体2: 葉（L08〜L15）
  L08・L09: 葉の幅（2座）
  L10: 葉の形（線形←→円形←→複葉）
  L11: 葉の縁（全縁←→深裂）
  L12: 葉の巻き込み（結球）
  L13: 葉の緑色素
  L14: 葉の赤紫色素
  L15: 葉の質感（光沢←→毛羽立ち←→多肉）
  L16: 葉の香り（なし←→強香）

染色体3: 花（L17〜L24）
  L17・L18: 花の赤色素（2座）
  L19・L20: 花の青色素（2座）
  L21: 花の黄色素
  L22: 花序の形（単頂/総状/穂状/散形/頭状）
  L23: 花弁の形（単弁/重弁/フリル/筒状）
  L24: 花の香り

染色体4: 果実・根（L25〜L32）
  L25: 果実の構造（液果/核果/仁果/豆果/集合果/なし）
  L26・L27: 果実の大きさ（2座）
  L28: 果実の色
  L29: 甘み・酸み
  L30: 苦み・辛み・旨み
  L31: 根の形態（細根/直根/塊根/球根/塊茎/根茎）
  L32: 根の大きさ・食用価値

染色体5: 生理・繁殖（L33〜L39）
  L33: 耐乾性
  L34: 耐寒性
  L35: 開花条件（長日/短日/中性/春化）
  L36: 成長速度
  L37: 種の数
  L38: 突然変異率
  L39: 毒性・特殊形質
```

### タンパク質15種
```
P00: 草丈制御     P01: 茎形態       P02: (予備)
P03: 葉形態       P04: 葉色素       P05: 葉表面
P06: 花赤色素     P07: 花青黄色素   P08: (予備)
P09: 花香り       P10: 果実形態     P11: 果実味
P12: 根形態       P13: 環境耐性     P14: 繁殖制御
```

### wiring（サーバー側のみ・非公開）
```javascript

  [0,  [0,1],        'multiply'],  // P00 草丈
  [1,  [2,3],        'additive'],  // P01 茎形態

  [4,  [13,14],      'inhibit'],   // P04 葉色素
  [5,  [12,16],      'additive'],  // P05 葉表面
  [6,  [17,18],      'multiply'],  // P06 花赤色素
  [7,  [19,20,21],   'additive'],  // P07 花青黄色素
  [9,  [24],         'simple'],    // P09 花香り
  [10, [26,27,28],   'additive'],  // P10 果実形態
  [11, [29,30],      'additive'],  // P11 果実味
  [12, [32],         'simple'],    // P12 根形態
  [13, [33,34],      'additive'],  // P13 環境耐性

## 成長システム（確定済み）

### 8段階
```
① 種子      → 形・色・大きさだけ見える
② 発芽      → 双子葉か単子葉かわかる
③ 幼植物    → 葉の形・色が見えてくる
④ 栄養成長期 → 草丈・茎色・草姿がわかる
⑤ 花芽形成  → 蕾の色・形がうっすらわかる
⑥ 開花      → 花の全貌判明。新種判定・命名タイミング
⑦ 結実      → 果実・種が形成される。収穫可能
⑧ 枯死・休眠 → 一年草は枯れる。多年草は休眠
```

### 通信設計
```
植えるとき（1回）:    lociを送信。germinationThresholdを受け取る
定期同期（1時間ごと）: params送信。health・progress・stageを受け取る
段階変化時:           定期同期に含まれる。説明文も受け取る
```

### 説明文の自動生成
```
説明文 = 基本描写（遺伝子から・固定）
       + 状態修飾（環境パラメータから・変動）
       + 異変描写（ストレス・健康度低下時）
```

## 生育条件と失敗（確定済み）

```
最適ゾーン:    健康に成長
ストレスゾーン: 成長が遅くなる（回復可能）
枯死ゾーン:    不可逆ダメージ（根腐れ・凍死・熱害・肥料焼け）

枯れても自動で消えない。プレイヤーが能動的に撤去する。
```

## 図鑑システム（確定済み）

```
自分の図鑑（ローカル）:     自分の観察記録・命名
他プレイヤーの図鑑（Firestore）: 公開した記録が見える
公式図鑑（管理者）:         新種申請を審査・承認した種のみ
```

### 新種申請フロー
```
開花 → 公式図鑑で検索 → 似たものがなければ申請
→ 申請時だけサーバーが遺伝的距離を計算
→ 管理者が審査・承認 → 公式図鑑に登録
```

- 基本的な描画は動作する

### 作り込み予定順序
```
Step 1: 花（花弁の形状・重なり・グラデーション・光沢）← 次のタスク

Step 3: 葉（葉脈・縁のギザギザ・光沢）

```

花:   inflo・petalShape・petalN・flowerSz・fR・fB・fY
果実: fruit・fruitSz・fruitN・frR・frY
段階: stage（0〜7）
```
